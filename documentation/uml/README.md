# Budget Analyser UML Diagrams

This directory contains PlantUML sources and generated images for the current SRC‑based architecture.

What’s here:
- class_diagram.puml and other .puml files that describe key packages and relationships
- Exported images (PNG/SVG) generated by the documentation update script

Generate diagrams:
- Easiest: run scripts/update_documentation.sh (requires Java, Graphviz)
- Manual: java -jar plantuml.jar <file.puml>
- In IDE: use a PlantUML plugin to preview .puml files directly

Prerequisites (for local generation):
- Graphviz (dot)
- Java (for PlantUML)

Architecture overview (matches src/ layout):
- Presentation (views)
  - LoginWindow and DashboardWindow shells
  - Pages: yearly_summary_page, earnings_page, expenses_page, payments_page, mapper_page, settings_page, upload_page
- Presentation (controller)
  - yearly_summary_stats_controller, earnings_stats_controller, expenses_stats_controller,
    payments_reconciliation_controller, mapper_controller, settings_controller
- Domain
  - statement_formatters (base_statement_formatter, citi_statement_formatter, discover_statement_formatter, default_statement_formatter)
  - transaction_processing / transaction_processor, reporting, category_mappers, protocols, errors
- Infrastructure
  - ini_config, json_mappings, statement_repository, column_mappings

Entrypoint and flow:
- Start the app with: python -m budget_analyser
- Flow: Login → Dashboard (header, sidebar) → select a page → controllers shape data → views render tables/trees

Troubleshooting
- Verify Graphviz installation: dot -v
- If IDE preview fails, export via the PlantUML JAR or use the online server: https://www.plantuml.com/plantuml/uml/
